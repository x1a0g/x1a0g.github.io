java实现BIO

  1. BIO服务端特点：
    1. 使用ServerSocket和Socket进行网络通信
    2. accept()方法阻塞等待新连接
    3. 每个客户端连接需要独立的处理线程
    4. 读写操作(readLine()/println())都是阻塞的
  2. 执行流程：
    1. 启动服务端后进入accept阻塞状态
    2. 客户端连接时创建新线程处理
    3. 线程内循环读取客户端输入并发送响应
    4. 客户端断开连接后线程结束
  3. 与NIO的主要区别：
    1. 每个连接占用独立线程
    2. 线程在IO操作时会被阻塞
    3. 无法高效处理大量并发连接
    4. 编程模型相对简单直接

java实现nio

  1. 服务器端流程：
    1. 创建Selector和ServerSocketChannel
    2. 绑定端口并注册ACCEPT事件
    3. 通过事件循环处理连接和读写操作
    4. 收到消息后发送回显响应
  2. 客户端流程：
    1. 创建SocketChannel并连接服务器
    2. 发送测试消息并等待响应
    3. 使用非阻塞模式进行基本IO操作
  3. 核心NIO组件：
    1. Selector：实现多路复用，监控多个通道的事件
    2. ByteBuffer：数据读写容器（注意flip操作）
    3. Channel：通信通道（ServerSocketChannel/SocketChannel）
